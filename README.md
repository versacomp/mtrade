# MTrade

A trading analysis tool that connects to the tastytrade REST API and DXLink real-time streaming service. Built with Flet (Python/Flutter) and designed to run as a desktop application.

## Features

- **OAuth2 / sandbox login** — authenticates against tastytrade's sandbox environment (`api.cert.tastyworks.com`)
- **Live market dashboard** — real-time quotes for major US indices (S&P 500, Nasdaq-100, Dow Jones)
- **Interactive candlestick chart** — any symbol, 1-minute candles, pan and zoom via mouse/touch
- **Institutional Liquidity view** — real-time signal detection across 40+ futures instruments with simulated trade management
- **Strategy Analysis view** — KPI dashboard, equity curve, and back-test engine over cached history
- **DXLink streaming** — WebSocket candle feed with exponential back-off reconnection and transparent demo fallback
- **Persistent candle cache** — 48-hour rolling cache per symbol stored in `~/.mtrade/cache/`

## Setup

### 1. Create a virtual environment and install dependencies

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
# or: source .venv/bin/activate  # Linux/macOS

pip install -r requirements.txt
```

### 2. Configure tastytrade credentials

Log in with your tastytrade **sandbox** username and password directly in the app UI on first launch. OAuth2 refresh tokens are supported via the `api/oauth.py` module if you have an OAuth application configured.

### 3. Run the app

```bash
python main.py
```

The app opens as a native desktop window. To run in a browser instead, change `view=ft.AppView.FLET_APP` to `view=ft.AppView.WEB_BROWSER` in `main.py`.

## Institutional Liquidity Strategy

The `/liquidity` view implements an **Institutional Liquidity Grab Reversal** strategy on 1-minute futures candles. The premise is that institutions intentionally push price through visible swing highs and lows to fill large orders against retail stop clusters, before reversing hard in the opposite direction.

### Instruments

40+ futures instruments across six sectors are supported natively with accurate price scaling:

| Sector | Symbols |
|---|---|
| Equity Index | MES, ES, MNQ, NQ, MYM, YM, M2K, RTY |
| Metals | MGC, GC, MSI, SI, HG, PL, PA |
| Energy | MCL, CL, NG, HO, RB |
| Interest Rates | ZB, ZN, ZF, ZT, SR3 |
| FX | 6E, 6J, 6B, 6A, 6C, 6S, 6N, 6M |
| Agricultural / Livestock | ZC, ZW, ZS, ZM, ZL, ZO, KC, CT, SB, CC, LE, GF, HE |

Quick-access chips for the most-traded micro contracts (MES, MNQ, M2K, MYM, MGC) sit at the top of the view. Any symbol in the registry can be loaded via the text field.

### Signal Detection

**Step 1 — Identify swing structure**

Swing highs and swing lows are identified using a 3-candle lookback on each side. Only swings from the most recent 30 candles are considered "live" for signal purposes.

**Step 2 — Detect the liquidity grab**

A grab candle must satisfy all three conditions simultaneously:

- **Bearish grab (Sell signal):** the wick sweeps *above* a prior swing high, but the candle *closes back below* it. The close-back distance (reversal) must be ≥ 30% of the wick above the level.
- **Bullish grab (Buy signal):** the wick sweeps *below* a prior swing low, but the candle *closes back above* it. The reversal must be ≥ 30% of the wick below the level.

Key institutional levels (4-hour high/low and previous day high/low) are scanned in addition to swing structure, producing tagged signals (`4HH`, `4HL`, `PDH`, `PDL`).

### Signal Quality Tiers

Every detected grab is independently assessed across two confirmation filters and assigned a visual tier:

| Tier | Criteria | Arrow style |
|---|---|---|
| **Prime** | RSI divergence confirmed **+** aligned with SMA 200 trend | Bright, large filled arrow |
| **Filtered** | RSI divergence confirmed, counter-trend | Bright hollow (stroke) arrow |
| **Weak** | No RSI divergence | Dim, small filled arrow |

**RSI Divergence** (Wilder's RSI-14): a bullish grab is divergent when the grab candle makes a price lower-low while RSI makes a higher-low relative to any candle in the prior 30 bars. Bearish divergence is the inverse. Divergence dots are drawn on the RSI sub-panel at each confirmed candle.

**Pro-Trend Filter**: the signal direction must agree with the SMA 200 macro trend — bullish signals require `close > SMA 200`; bearish signals require `close < SMA 200`. When enabled, only Prime (Tier 1) signals open simulated trades. The chart background is tinted green when above SMA 200 and red when below.

**Range Filter (ADX)**: ADX(14) < 25 indicates a ranging, mean-reversion environment. The grabbed level must also fall within 25% of the rolling 20-period high or low boundary. This confirms the trade is a genuine range rotation from a structural boundary rather than a mid-range noise event.

### Chart Overlays

The candlestick canvas renders the following overlays simultaneously:

- **SMA 50** — orange line
- **SMA 200** — green (uptrend) or red (downtrend); determines chart background tint
- **20-period Range Bands** — rolling high/low curves in dark teal; mark the current range boundaries used by the range filter
- **Key Levels** — dashed gold lines (PDH/PDL) and gray lines (4H H/L)
- **Volume Profile** — left-anchored proxy volume bars with the Point of Control highlighted in orange
- **Signal arrows** — per-tier sizing and color as described above
- **RSI sub-panel** — Wilder RSI(14) with OB(70)/OS(30) reference lines and divergence dots
- **ADX sub-panel** — overlaid in the same RSI panel; cyan when ranging (< 25), amber when trending (≥ 25); dashed threshold line at 25

Pan (click-drag) and zoom (scroll wheel) are fully interactive. The viewport can be locked to live edge or scrolled back through the 4-hour buffer.

### Simulated Trade Management

Every Prime-tier signal that passes all active filters opens a single simulated trade:

| Parameter | Rule |
|---|---|
| Entry | Signal candle close price |
| Stop loss | Signal candle wick tip (low for bull, high for bear) |
| Take profit | Entry ± risk × 2.0 (1:2 R:R) |
| Max concurrent trades | 1 |

**Ratcheting stop** — once in a trade, the stop is automatically promoted through two stages:

| Stage | Trigger | Stop moves to |
|---|---|---|
| 0 → Breakeven | Price reaches 50% of TP distance | Entry price (zero-risk) |
| 1 → Trailing | Price reaches 75% of TP distance | Peak price − risk × 0.5, ratcheted |

The stop only ever moves in the favorable direction. P&L reflects the *actual* stop price at close, so a breakeven exit records 0 pts and a trailing-stop exit records a positive result.

**Opposing signal flip** — an opposing Prime signal closes the current position at market and starts a 120-second cooldown. After the cooldown, re-entry viability is checked (price must still be on the correct side of the grabbed level and trend alignment must still hold) before a re-entry trade is opened. Re-entry trades are tagged with `+RE` in the source field.

Trade levels (entry, stop, take profit) are drawn as dashed horizontal lines on the chart. The stop line color changes from red (original) → amber (breakeven) → green (trailing) to reflect the current protection stage.

### Alert System

When a new signal is detected on the active symbol:
- An overlay badge flashes in the top-right corner of the chart
- A snack-bar notification appears at the bottom of the screen
- An audible beep sounds (Windows only, via `winsound`)

Individual alert types can be toggled via the **Alert** and **Sound** checkboxes. The three strategy filters (**Trend Filter**, **Range Filter**) are also toggleable at runtime without restarting the stream.

## Strategy Analysis View

The `/analysis` view computes performance statistics from all persisted simulated trades for the selected symbol, and provides a full back-test over the cached 48-hour candle history.

### Live Trade KPIs

| Metric | Description |
|---|---|
| Win Rate | % of closed trades that were profitable |
| Profit Factor | Gross wins ÷ gross losses |
| Expectancy | Expected pts per trade (win rate × avg win + loss rate × avg loss) |
| Max Drawdown | Largest peak-to-trough equity decline |
| Sharpe | Mean P&L ÷ std dev of P&L per trade |
| Recovery Factor | Total P&L ÷ max drawdown |
| W:L Ratio | Average winning trade ÷ average losing trade |
| Total P&L | Cumulative points across all closed trades |

An equity curve canvas shows the running P&L per trade, colored green/red by sign, with peak and trough markers. A signal source breakdown table and a ratcheting-stop protection breakdown card show how exits were distributed.

### Back-test Engine

The back-test replays the full strategy over the cached history with no look-ahead bias — each candle only sees swing structure that existed before it. The **Trend filter** and **Range filter** checkboxes mirror the live view toggles and can be independently enabled or disabled to compare results.

## Data Storage

All persistent data lives under `~/.mtrade/`:

```
~/.mtrade/
├── cache/
│   ├── candles/        # Per-symbol 48-hour candle cache (JSON, atomic writes)
│   └── sim_trades/     # Per-symbol simulated trade history (JSON, atomic writes)
```

The candle cache is seeded immediately on symbol activation so the chart shows historical data before the DXLink stream connects. It is flushed to disk in a background thread at most once every 30 seconds per symbol.

## Streaming Architecture

Market data arrives via **DXLink** — tastytrade's WebSocket-based market data protocol.

**Connection lifecycle:**
1. Fetch a short-lived quote token via `GET /api-quote-tokens`
2. Open WebSocket → `SETUP` → `AUTH` → `CHANNEL_REQUEST` → `FEED_SUBSCRIPTION`
3. Candle events arrive in `COMPACT` format (flattened field arrays)
4. `KEEPALIVE` frames sent every 30 seconds to keep the connection alive

**Resilience:**
- Exponential back-off retry (5s base, 60s ceiling, ±10% jitter, up to 10 attempts)
- A healthy stream that runs for ≥ 60 seconds resets the attempt counter
- After all retries are exhausted, the view automatically falls back to deterministic demo data so the chart remains usable without a live connection
- The connection status dot in the app bar reflects `LIVE`, `RECONNECTING`, `OFFLINE`, or `DEMO` in real time
